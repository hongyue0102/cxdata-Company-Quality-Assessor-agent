# 执行标准程序 (Recommended Workflow)

当用户发起查询时，AI 推荐按照以下步骤执行：

1. **Step 1: 调用统一工具**
- 使用 `python ./scripts/api_query.py <API_ID> key=value [key=value] ...` 命令获取接口数据
- 工具会自动完成：加载配置 → 获取token → 发送请求 → 解压数据 → 输出JSON结果
- **调用方式**：所有接口统一使用 key=value 格式传参

2. **Step 2: 解析结果**
- 工具输出为格式化的JSON数据，直接解析并呈现给用户

# 数据处理说明

## 配置说明
本技能所有接口需要认证token，配置方式：
1. 在 `./scripts/.env` 文件中配置 `USER_KEY` 和 `BASE_URL`
2. 调用 `./scripts/api_query.py` 时会自动获取并使用token

## 脚本功能
`./scripts/api_query.py` 脚本功能：
- 自动加载 `./scripts/.env` 配置文件
- 自动获取 `authtoken`
- 构建请求URL并发送GET请求
- 自动处理 Base64 解码和 Gzip 解压
- 输出格式化的JSON结果

# 使用场景

- 用户查询A股首次发行上市的相关信息，范围涵盖意向书、发行公告、发行结果、上市公告相关进程情况。主要信息包括：发行方式、发行价、网上和网下申购起始日、网上和网下申购配售情况、计划发行量、实际发行量、中签率、募集资金、发行费用、发行上市相关中介机构等
- 用户查询公司首发配售明细数据。主要信息包括：公告日期、配售类别、配售对象名称、获配股数、投资者名称等
- 用户查询B股首次发行上市的相关信息，范围涵盖意向书、发行公告、发行结果、上市公告相关进程情况。主要信息包括：发行方式、发行价、网上和网下申购起始日、网上和网下申购配售情况、计划发行量、实际发行量、中签率、募集资金、发行费用、发行上市相关中介机构等
- 用户查询上市公司历次增发（包括公开、非公开）机构配售明细。主要信息包括：公告日期、配售类别、配售机构名称、获配股数等
- 用户查询上市公司历次增发（包括公开、非公开）审核的信息。主要信息包括：审核结果公告日、审核是否通过、委员会类型等
- 用户查询上市公司历次增发（包括公开、非公开）的发行及实施信息，范围涵盖增发的基本方案、日期安排、发行结果。主要信息包括：增发类型、发行日期起始日、股权登记日、增发股份上市日期、发行对象、网上申购起始日、发行价、实际发行量、发行费用、发行上市相关中介机构等
- 用户查询上市公司历次增发（包括公开、非公开）预案及变动情况。主要信息包括：增发类型、预案有效期起始日、计划发行数量、计划发行价、募集资金用途等
- 用户查询上市公司历次配股证监会审核的信息。主要信息包括：审核结果公告日、审核是否通过、委员会类型等
- 用户查询上市公司历次配股的大股东认配明细。主要信息包括：股东名称、股东代码、认配方式、配售股数等
- 用户查询上市公司历次配股的发行及实施信息。主要信息包括：配股说明书公告日期、配股股本基数、股权登记日、除权日、配股对象、配股价、配股比例、配售数量、配股募集资金、配股上市日期、配股相关中介机构等
- 用户查询股票首发审核进展情况，范围涵盖审核召开公告及审核结果公告信息。主要信息包括：公司名称、股票代码、股票简称、审核会议召开日、审核结果公告日、审核是否通过、委员会类型（发审委、上市委等）、核准发行股数、计划发行量等
- 用户查询上市公司历次分红方案实施情况，范围涵盖年度、中期、临时分红实施等。主要信息包括：截止日期、分红股本基数、分红实施内容、股权登记日、除权除息日、每10股/份送股、每10股/份转增股、每10股/份派现、送转后总股本、股权登记日收盘价、全体股东分红总额、送转股上市日、股息到帐日期等
- 用户查询公司历次分红预案、预案变动及股东大会审核的情况。主要信息包括：截止日期、预案公布日、股东大会决议公告日、预案是否通过、每10股/份送股、每10股/份转增股、每10股/份派现、方案变动说明、分红方案等
- 用户查询上市公司历次配股预案及变动情况。主要信息包括：预案有效期起始日、计划配股比例、股东大会决议公告日期等
- 用户查询上市公司分季度按行业统计的分红与盈利情况。主要信息包括：行业名称、行业分类体系、截止日期、行业内公司家数、盈利家数、亏损家数、分红家数、盈利公司股利支付率等
- 用户查询公司历次融资事件的事件清单，便于首发、增发、配股等融资系列表根据事ID进行串联。主要信息包括：机构中文名称、股票类别参数、事件标题、事件概述、拟上市证券市场、拟上市板块等
- 用户查询公司上市前的董事会、监事会、经营层主要职务的人员任职情况。主要信息包括：公司名称、人物姓名、职位名称、任职起始日、任职截止日等
- 用户查询公司发行前的主要股东持股情况。主要信息包括：公司名称、截止日期、股东名称、持股排名、持股数等
- 用户查询公司公司发行前历次申报及审核状态变动情况。主要信息包括：受理日期、公司全称、注册地、证监会行业、审核状态、更新日期、保荐机构等
- 用户查询上市公司历次配股的承销商的信息。主要信息包括：承销商类型、承销商名称、承销数量等
- 用户查询首发时承销商的信息。主要信息包括：招股公告书日期、承销商类型、承销商名称、承销数量等
- 用户查询中国存托凭证(CDR)与其基础证券的转换信息。主要信息包括：存托凭证份数、基础股票股数、基础股票与CDR之间的转换比例等
- 用户查询股票发行人的相关信息。主要信息包括：股票代码、股票简称、招股日期、公司名称、注册资本、招股时注册资本、上市时注册资本、上市状态、上市时董事会人数、职工人数等
- 用户查询公司辅导备案、境外上市、挂牌审核以及定向发行审核等项目动态。主要信息包括：公司名称、所属地区、备案审核类型、备案审核状态、审核通过日、拟上市证券交易所、辅导机构等
- 用户查询上市公司首次公开发行股票招股申报稿中披露的计划发行信息。主要信息包括：公司名称、公告日期、计划发行量、发行方式、股票类型等
- 用户查询公开增发与非公开增发时承销商的信息。主要信息包括：增发类型、招股公告书日期、承销商类型、承销商名称等

```bash
python ./scripts/api_query.py <API_ID> key=value [key=value] ...
```

## 故障排除

- **调用失败**：检查 `./scripts/.env` 文件是否存在并正确配置了 `CXDA_USER_KEY` 和 `BASE_URL`，确认网络连接正常，token未过期。
- **输出为空**：确认输入参数是否正确，检查API_ID是否匹配查询类型。
- **权限问题**：接口返回无权限或者权限到期时，提示用户前往`https://yun.ccxe.com.cn/`联系客服
- **scripts路径问题**：文档中描述的是skill相对路径，请求时确认路径是否正确
- **相同含义字段说明**：ORG_UNI_CODE==COM_UNI_CODE;STK_UNI_CODE==BOND_UNI_CODE；如无明确说明股票代码不需要携带SH、HK等交易所代码
- **接口输入字段不明确**：接口输入参数是ORG_UNI_CODE、STK_UNI_CODE等参数时可以先通过对应的基础接口调用尝试获取，获取不到时反馈并停止
- **其他异常**：出现其他异常时停止输出并返回异常原因
