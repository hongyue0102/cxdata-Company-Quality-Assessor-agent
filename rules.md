# rules.md - 硬性规则

## 分析纪律

1. **必须有数据才能分析**：如果 fetch_data.py 返回关键数据缺失，必须明确告知用户，不能凭空编造
2. **不编造财务数据**：报告中出现的所有财务数据、排名、持股比例必须来自分析数据，不得自行估算或编造
3. **不编造行业对比**：如果数据中没有行业对比信息，标注"数据不足，无法对比"，不得自行编造
4. **不使用外部信息**：所有分析只基于提供的数据，不引用数据以外的网络信息
5. **评分使用100分制**：final_score 即为100分制得分，不需要换算
6. **等级划分固定**：A≥90、B≥75、C≥60、D≥40、E<40，不得随意调整

## 数值固定（防止两次报告不一致）

7. **禁止篡改 Python 计算的数值**：所有 `final_score` / `grade` / 各模块评分 / 行业分位 / 财务比率 / 治理指标 / 风险扣分 必须原样使用 `analysis.json` 输出，不得修改、估算、重新计算或换算单位
8. **禁止修改 generate_report.py 输出的骨架结构**：表格行/列、字段名、字段顺序、单位（亿/元/%/次/条/名）必须与脚本输出完全一致，不得增删、重排或改写
9. **LLM 只在「**AI 解读：**」槽位填字**：所有文字解读只能写在脚本预留的槽位下方，不得跨槽位填写或修改其他段落
10. **禁止编造未在 analysis.json 中出现的数据**：包括但不限于未提供的财务指标、行业对比、个股名称、关联公司

## 免责声明

11. **所有报告必须附带免责声明**：本报告基于公开财务数据推演，不构成任何投资建议
12. **不能给出明确的买入/卖出操作建议**
13. **禁止省略或修改报告末尾的免责声明文本**

## 数据安全

14. **不暴露 API KEY**：报告中不包含任何密钥、token 信息；`auth.py status` 输出的 `CXDA_USER_KEY` 已脱敏，不要原样展示给用户
15. **不记录用户敏感信息**

## 报告格式

16. **使用八段式模板**：总评分 → 商业模式 → 行业地位 → 成长质量 → 财务质量 → 治理配置 → 扣分项 → 结论
17. **数据必须用表格呈现**：财务指标、收入结构、年度增长等
18. **日期格式统一使用 YYYY-MM-DD**
